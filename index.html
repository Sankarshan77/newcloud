The Foundation

  • On-demand Self-service: Automatic provisioning without human interaction.
  • Broad Network Access: Standard mechanisms (mobile, web).
  • Resource Pooling: Multi-tenant physical/virtual resources.
  • Rapid Elasticity: Scaling instantly with demand.
  • Measured Service: Metered billing (Pay-as-you-go).
Architecture

The SPI Model

IaaS, PaaS, SaaS

Service Delivery

IaaS: Infrastructure. You manage OS, Runtime, and Data. (e.g., Azure VMs).

PaaS: Platform. You manage only Apps and Data. Provider manages OS. (e.g., AWS Elastic Beanstalk).

SaaS: Software. Provider manages the whole stack. (e.g., Google Workspace).

Deployment

Public vs Private vs Hybrid

Where is it hosted?

Cloud Environments

Public: Multi-tenant, shared over internet. Cost-effective.

Private: Single-tenant, high security. On-premise or hosted.

Hybrid: Integration of Public + Private. Used for "Cloud Bursting" during peak loads.

Community: Shared by orgs with common goals (e.g., Government or Healthcare).

Technology

Hypervisors & Virtualization

The Engine of Cloud

Virtualization Stack

Hypervisor (VMM): Software that creates and runs VMs.

Type 1 (Bare Metal): Runs directly on hardware. Fast/Secure. (Xen, ESXi).

Type 2 (Hosted): Runs on a Host OS. Good for testing. (VirtualBox).

Finance

CapEx vs OpEx

Financial Shift

Cloud Economics

CapEx: Capital Expenditure. Upfront costs for hardware. Rigid.

OpEx: Operational Expenditure. Monthly bills based on use. Flexible.

TCO: Total Cost of Ownership. Includes cooling, power, and space costs saved by moving to Cloud.

Security

Shared Responsibility

The Boundary Line

Security Ownership

Provider: Security OF the Cloud (Physical, Hardware, Global Infra).

Customer: Security IN the Cloud (Data, IAM, Patching Guest OS, Encryption).

Architecture

Horizontal vs Vertical Scaling

How to grow?

Scalability Types

Vertical (Scale Up): Adding more RAM/CPU to an existing server. Has a physical limit.

Horizontal (Scale Out): Adding more instances/servers. Infinite potential (Elasticity).

Availability

DR and Business Continuity

RTO and RPO

Disaster Recovery

RTO (Recovery Time Objective): Target time to restore service.

RPO (Recovery Point Objective): Maximum age of files that must be recovered (Data Loss Tolerance).

High Availability: Ensuring 99.9% (Three Nines) or more uptime.
