Cloud Computing Mastery Deck

Fundamentals

5 NIST Characteristics

The "Definition" of Cloud

  • On-demand Self-service: Provisioning without human interaction with provider.
  • Broad Network Access: Available over the net via standard mechanisms (mobile, laptops).
  • Resource Pooling: Multi-tenant model; physical/virtual resources assigned dynamically.
  • Rapid Elasticity: Scaling in/out quickly to match demand.
  • Measured Service: Usage is monitored, controlled, and reported (Pay-per-use).
Architecture

SPI Model Details

SaaS, PaaS, IaaS

IaaS: You manage Guest OS, Middleware, Data, and Apps. Provider manages Hardware/Virtualization.

PaaS: You manage only Apps and Data. Provider manages the OS and Runtime environment.

SaaS: Provider manages everything. You simply use the interface (Browser/App).

Tech

Hypervisors (VMM)

Type 1 vs Type 2

Type 1 (Bare Metal): Runs directly on hardware. Highly efficient. Used in Data Centers. (e.g., Xen, KVM, ESXi).

Type 2 (Hosted): Runs on top of a Host OS. Used for dev/testing. (e.g., VMware Workstation, Oracle VirtualBox).

Security

Shared Responsibility

The Boundary Line

Security OF the Cloud: Global infrastructure, Hardware, Software, Networking, and Facilities. (Provider's Job)

Security IN the Cloud: Client-side data, Identity & Access (IAM), OS configuration, Firewall rules. (User's Job)

Deployment

Cloud Types

Public, Private, Hybrid

Public: Resources owned by provider; shared by multiple orgs.

Private: Exclusive use by one org. Can be on-prem or off-prem.

Hybrid: Combines public/private; allows Cloud Bursting (moving load to public cloud during peaks).
